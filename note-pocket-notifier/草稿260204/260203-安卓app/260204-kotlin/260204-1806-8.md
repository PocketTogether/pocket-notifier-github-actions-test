# 📄 `.github/workflows/android.yml`

```yaml
name: Android CI

on:
  push:
    branches: [ main ]
  pull_request:

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
      # 拉取仓库代码
      - name: Checkout
        uses: actions/checkout@v4

      # 设置 JDK（Android Gradle Plugin 需要 17）
      - name: Set up JDK 17
        uses: actions/setup-java@v4
        with:
          distribution: temurin
          java-version: 17

      # 缓存 Gradle（加速构建）
      - name: Cache Gradle
        uses: actions/cache@v4
        with:
          path: |
            ~/.gradle/caches
            ~/.gradle/wrapper
          key: ${{ runner.os }}-gradle-${{ hashFiles('**/*.gradle*', '**/gradle-wrapper.properties') }}
          restore-keys: |
            ${{ runner.os }}-gradle-

      # 如果你使用 keystore（可选）
      # TODO: 如需签名，请在仓库 Secrets 中添加：
      # - ANDROID_KEYSTORE_BASE64
      # - ANDROID_KEYSTORE_PASSWORD
      # - ANDROID_KEY_ALIAS
      # - ANDROID_KEY_ALIAS_PASSWORD
      #
      # 然后取消注释以下步骤：
      #
      # - name: Decode keystore
      #   run: |
      #     echo "$ANDROID_KEYSTORE_BASE64" | base64 --decode > app/release.keystore
      #
      # 并在 app/build.gradle.kts 中配置 signingConfigs

      # 赋予 gradlew 可执行权限
      - name: Grant execute permission for gradlew
        run: chmod +x gradlew

      # 构建 Release APK
      - name: Build Release APK
        run: ./gradlew assembleRelease

      # 上传构建产物
      - name: Upload APK
        uses: actions/upload-artifact@v4
        with:
          name: PocketNotifier-APK
          path: app/build/outputs/apk/release/*.apk
```

# ✅ 说明与可修改位置

### ✔ 使用 `actions/setup-java@v4`

### ✔ 使用 `./gradlew assembleRelease`

构建 Release APK。

### ✔ 输出 APK 到 artifacts

路径：

```
app/build/outputs/apk/release/*.apk
```

